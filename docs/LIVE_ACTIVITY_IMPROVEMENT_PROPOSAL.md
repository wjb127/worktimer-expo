# Live Activity 잠금화면 UI 개선안

## 현재 상태

### 표시 정보
```
┌─────────────────────────────────────┐
│  업무 중                            │
│  오늘 총: 2:30:45                   │
│  ████████████░░░░░░░░  (진행 막대)  │
└─────────────────────────────────────┘
```

### 현재 API 사용
```typescript
const state = {
  title: '업무 중',
  subtitle: `오늘 총: ${formatTime(todayTotal + elapsedSeconds)}`,
  progressBar: { progress: (elapsedSeconds % 3600) / 3600 },
};
```

### 한계점
- 정보가 2가지(상태, 오늘 총 시간)로 제한적
- 현재 세션 경과 시간을 직관적으로 확인하기 어려움
- 진행 막대의 의미가 모호함 (1시간 기준 반복)

---

## 개선안 목록

### 안 1: 현재 세션 시간 강조
**콘셉트**: 현재 진행 중인 세션의 경과 시간을 메인으로 표시

```
┌─────────────────────────────────────┐
│  현재 세션: 1:23:45                 │
│  오늘 누적: 5:30:00                 │
│  ████████████░░░░░░░░  (진행 막대)  │
└─────────────────────────────────────┘
```

**장점**
- 현재 얼마나 일했는지 바로 확인 가능
- 오늘 총 시간도 함께 파악 가능

**단점**
- 두 줄의 시간 정보로 약간 복잡해 보일 수 있음

---

### 안 2: 시작 시간 표시
**콘셉트**: 언제 시작했는지 표시하여 자연스럽게 경과 시간 유추

```
┌─────────────────────────────────────┐
│  09:30부터 업무 중                  │
│  오늘 총: 5:30:00                   │
│  ████████████░░░░░░░░  (진행 막대)  │
└─────────────────────────────────────┘
```

**장점**
- 시작 시간을 명확히 알 수 있음
- 간결한 표현

**단점**
- 경과 시간을 직접 계산해야 함

---

### 안 3: 목표 시간 대비 진행률
**콘셉트**: 일일 목표 시간 대비 진행 상황 표시

```
┌─────────────────────────────────────┐
│  업무 중 (5h 30m / 8h 목표)         │
│  현재 세션: 1:23:45                 │
│  ████████████████░░░░  (68%)        │
└─────────────────────────────────────┘
```

**장점**
- 목표 달성률을 직관적으로 확인
- 동기 부여 효과

**단점**
- 목표 시간 설정 기능 추가 필요
- 복잡도 증가

---

### 안 4: 심플 듀얼 타이머
**콘셉트**: 두 개의 시간을 명확하게 구분 표시

```
┌─────────────────────────────────────┐
│  지금: 1:23:45    오늘: 5:30:00     │
│  ████████████░░░░░░░░  (진행 막대)  │
└─────────────────────────────────────┘
```

**장점**
- 한 줄에 두 정보를 간결하게 표시
- 공간 효율적

**단점**
- 글자가 작아질 수 있음

---

### 안 5: 상태 + 경과 시간 중심
**콘셉트**: title에 경과 시간, subtitle에 추가 정보

```
┌─────────────────────────────────────┐
│  1:23:45 업무 중                    │
│  오늘 총 5시간 30분 달성            │
│  ████████████░░░░░░░░  (진행 막대)  │
└─────────────────────────────────────┘
```

**장점**
- 가장 중요한 현재 경과 시간이 눈에 띔
- 부가 정보는 자연스럽게 subtitle로 분리

**단점**
- 없음

---

### 안 6: 이번 주 통계 포함
**콘셉트**: 오늘과 이번 주 정보 함께 표시

```
┌─────────────────────────────────────┐
│  업무 중 1:23:45                    │
│  오늘 5h 30m · 이번 주 32h          │
│  ████████████░░░░░░░░  (진행 막대)  │
└─────────────────────────────────────┘
```

**장점**
- 주간 업무량도 함께 파악 가능
- 장기적 동기 부여

**단점**
- 주간 통계 계산 로직 추가 필요
- 정보가 많아 복잡해 보일 수 있음

## 추천안

### 1순위 추천: 안 5 (상태 + 경과 시간 중심)

```typescript
const state = {
  title: `${formatTime(elapsedSeconds)} 업무 중`,
  subtitle: `오늘 총 ${formatDuration(todayTotal + elapsedSeconds)} 달성`,
  progressBar: { progress: (elapsedSeconds % 3600) / 3600 },
};
```

**추천 이유**
1. **직관성**: 잠금화면을 봤을 때 "지금 얼마나 일했는지"가 바로 보임
2. **간결함**: expo-live-activity API 내에서 구현 가능, 추가 기능 불필요
3. **동기부여**: "달성"이라는 표현으로 긍정적 피드백
4. **구현 용이성**: 현재 코드에서 title/subtitle 문자열만 수정하면 됨

## 추가 개선 고려사항

### 진행 막대 개선
현재 진행 막대는 `(elapsedSeconds % 3600) / 3600`으로 1시간마다 리셋됩니다.

**대안**
1. **8시간 기준**: 하루 목표 8시간 대비 진행률
   ```typescript
   progress: Math.min((todayTotal + elapsedSeconds) / (8 * 3600), 1)
   ```

2. **30분 단위**: 더 자주 업데이트 느낌
   ```typescript
   progress: (elapsedSeconds % 1800) / 1800
   ```

3. **진행 막대 제거**: 시간 정보만으로 충분할 수 있음

---

### Dynamic Island 고려
- expo-live-activity는 잠금화면과 Dynamic Island를 동일하게 처리
- Compact/Minimal 모드에서는 title만 표시될 수 있음
- title에 가장 중요한 정보(경과 시간)를 배치하는 것이 중요
